# AG Grid + Material UI (MUI) 학습 정리

---

## 1. AG Grid

스프레드시트처럼 데이터를 표시하는 React용 데이터 그리드 컴포넌트.  
필터링, 정렬, 페이지네이션, 피벗 등의 기능을 제공한다.

### 설치
```bash
npm install ag-grid-community ag-grid-react
```

> 버전 불일치 발생 시 해결 방법:
> 1. `package.json`에서 ag-grid 관련 항목 삭제
> 2. `node_modules`에서 ag-grid 관련 폴더 삭제
> 3. `node_modules/.vite` 폴더 삭제 (vite 캐시 제거)
> 4. `npm install ag-grid-community ag-grid-react` 재실행

---

### 기본 사용법 (v35 기준)

```tsx
import { useState } from 'react';
import axios from 'axios';
import { AgGridReact } from 'ag-grid-react';
import { ColDef, ICellRendererParams } from 'ag-grid-community';
import { ModuleRegistry, AllCommunityModule, themeQuartz } from 'ag-grid-community';
import './App.css'

ModuleRegistry.registerModules([AllCommunityModule]); // v35 필수 등록

type Repository = {
  id: number;
  full_name: string;
  html_url: string;
};

function App() {
  const [keyword, setKeyword] = useState('');
  const [repodata, setRepodata] = useState<Repository[]>([]);
  const [columnDefs] = useState<ColDef[]>([
    { field: 'id', sortable: false, filter: true },
    { field: 'full_name', sortable: true, filter: true },
    { field: 'html_url', sortable: true, filter: true },
  ]);

  const handleClick = () => {
    axios.get<{ items: Repository[] }>(
      `https://api.github.com/search/repositories?q=${keyword}`
    )
    .then(response => setRepodata(response.data.items))
    .catch(error => console.log(error));
  }

  return (
    <div className='App'>
      <input type="text" onChange={e => setKeyword(e.target.value)} value={keyword} />
      <button onClick={handleClick}>검색</button>
      <div className="ag-theme-material" style={{ height: 500, width: 850 }}>
        <AgGridReact
          rowData={repodata}
          columnDefs={columnDefs}
          theme={themeQuartz}
        />
      </div>
    </div>
  )
}
```

### 구조 포인트

- `rowData` : 그리드에 표시할 데이터 배열. 객체 배열을 넣으면 **key → 컬럼 / value → 행** 으로 자동 매핑된다.
- `columnDefs` : 컬럼별 설정을 정의하는 상태. `ColDef[]` 타입으로 명시.
- `<AgGridReact>`는 반드시 크기가 지정된 `<div>`로 감싸야 한다 (라이브러리 구조상 필수).

---

### ColDef 주요 속성

| 속성 | 설명 |
|------|------|
| `field` | 데이터 객체의 key 이름 |
| `sortable` | 정렬 기능 활성화 여부 |
| `filter` | 필터 기능 활성화 여부 |
| `cellRenderer` | 셀 내용을 커스텀 JSX로 렌더링 |

> 전체 ColDef 속성 참고: https://www.ag-grid.com/react-data-grid/column-properties/

---

### cellRenderer — 셀 커스텀

기본 컬럼은 string 데이터를 그대로 표시하지만, `cellRenderer`를 사용하면 HTML 태그 삽입이 가능하다.  
예를 들어 `html_url`을 단순 텍스트가 아닌 실제 링크로 만들 수 있다.

```tsx
const [columnDefs] = useState<ColDef[]>([
  { field: 'id', sortable: false, filter: true },
  { field: 'full_name', sortable: true, filter: true },
  {
    field: 'html_url',
    cellRenderer: (params: ICellRendererParams) => (
      <a href={params.value} target="_blank" rel="noreferrer">
        {params.value}
      </a>
    )
  }
]);
```

> `params.value` → 해당 셀의 값 (여기서는 html_url 문자열)  
> `cellRenderer` 내부에서 버튼, a태그 등 임의의 JSX 작성 가능.

## 2. Material UI (MUI)

구글의 Material Design을 구현한 React 컴포넌트 라이브러리.  
버튼, 리스트, 테이블, 카드, 다이얼로그 등 다양한 컴포넌트를 제공한다.

### 설치
```bash
npm install @emotion/react@11.11.1
npm install @emotion/styled@11.11.0
npm install @mui/material@5.14.8
```

---

### 주요 컴포넌트 소개

#### 레이아웃

```tsx
import { Container, AppBar, Toolbar, Typography } from '@mui/material';

<Container>
  <AppBar position='static'>
    <Toolbar>
      <Typography variant='h6'>
        Shopping List
      </Typography>
    </Toolbar>
  </AppBar>
</Container>
```

| 컴포넌트 | 역할 |
|----------|------|
| `Container` | 콘텐츠를 가로 중앙 배치. `maxWidth` prop으로 최대 너비 지정 (기본값: `'lg'`) |
| `AppBar` | 상단 네비게이션 바 |
| `Toolbar` | AppBar 내부 레이아웃 컨테이너 |
| `Typography` | 텍스트 컴포넌트. `variant` prop으로 h1~h6, body1 등 지정 |

#### 버튼 variant 종류

```tsx
<Button variant="text">Text</Button>         // 텍스트만
<Button variant="outlined">Outlined</Button> // 테두리
<Button variant="contained">Contained</Button> // 채워진 버튼
```

---

### Dialog (모달) 컴포넌트

```tsx
import { Dialog, DialogTitle, DialogContent, DialogActions, Button, TextField } from '@mui/material';

const [open, setOpen] = useState(false);

<>
  <Button onClick={() => setOpen(true)}>Add Item</Button>

  <Dialog open={open} onClose={() => setOpen(false)}>
    <DialogTitle>New Item</DialogTitle>
    <DialogContent>
      {/* 입력 필드 */}
    </DialogContent>
    <DialogActions>
      <Button onClick={() => setOpen(false)}>Cancel</Button>
      <Button onClick={addItem}>Add</Button>
    </DialogActions>
  </Dialog>
</>
```

- `Dialog`의 `open` prop이 `true`일 때 모달이 표시된다.
- `onClose`는 배경 클릭 시 실행되는 콜백이다.
- `TextField`는 MUI 스타일이 적용된 `<input>`과 동일하게 작동한다.

### Shopping List 실습

#### App.tsx — 타입 및 상태 정의

```tsx
export type Item = {
  product: string;
  amount: string;
}
```

#### AddItem.tsx — 방법 1 (필드별 개별 핸들러)

```tsx
import { useState } from "react";
import { TextField, Dialog, DialogActions, DialogContent, DialogTitle, Button } from "@mui/material";
import { Item } from "../App";

export default function AddItem(props) {
  const [open, setOpen] = useState(false);
  const [item, setItem] = useState<Item>({ product: '', amount: '' });

  return (
    <>
      <Button onClick={() => setOpen(true)} variant="contained">Add Item</Button>
      <Dialog open={open} onClose={() => setOpen(false)}>
        <DialogTitle>New Item</DialogTitle>
        <DialogContent>
          <TextField
            value={item.product} label='Product' margin="dense" fullWidth
            onChange={e => setItem({ ...item, product: e.target.value })}
          />
          <TextField
            value={item.amount} label='Amount' margin="dense" fullWidth
            onChange={e => setItem({ ...item, amount: e.target.value })}
          />
        </DialogContent>
        <DialogActions>
          <Button onClick={() => setOpen(false)}>Cancel</Button>
          <Button onClick={() => props.addItem(item)}>Add</Button>
        </DialogActions>
      </Dialog>
    </>
  );
}
```

#### AddItem.tsx — 방법 2 (공통 handleChange 적용)

필드가 많을 때 유리한 방식. `e.target.name`을 활용한다.

```tsx
const handleChange = e => {
  setItem({ ...item, [e.target.name]: e.target.value });
}

// TextField에 name 속성 추가
<TextField
  value={item.product} label='Product' margin="dense" fullWidth
  onChange={handleChange} name='product'
/>
<TextField
  value={item.amount} label='Amount' margin="dense" fullWidth
  onChange={handleChange} name='amount'
/>
```

> `[e.target.name]` — 동적 key를 이용한 객체 업데이트 (이전 수업의 계산된 속성명 응용)

### Props Drilling — addItem 함수 전달

`addItem()` 함수는 App(상위)에 정의되어 있고, AddItem(하위) 컴포넌트에서 호출해야 한다.  
이때 함수를 prop으로 전달하며, 타입은 아래와 같이 정의한다.

```tsx
type AddItemProps = {
  addItem: (item: Item) => void;
}

export default function AddItem({ addItem }: AddItemProps) {
  // ...
}
```

| 위치 | 역할 |
|------|------|
| App (상위) | `items` 상태 관리, `addItem()` 함수 정의 |
| AddItem (하위) | 모달 UI, 입력 상태 관리, prop으로 받은 `addItem()` 호출 |

> `open` 상태와 `setOpen()`은 AddItem 내부에서만 쓰이므로 하위 컴포넌트에 두면 된다.
