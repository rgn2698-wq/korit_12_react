## 1. 환경 변수 설정

API URL이 로컬 → 배포 환경으로 바뀔 때마다 코드를 수정하는 것은 비효율적이다.  
환경 변수로 분리하면 `.env` 파일 한 곳만 수정하면 된다.

### 설정 방법

```env
# 루트 폴더 .env 파일
VITE_API_URL=http://localhost:8080
```

> Vite 프로젝트의 환경 변수는 반드시 `VITE_`로 시작해야 한다.

### 사용 방법

```ts
// import.meta.env.변수명 으로 참조
axios.get(`${import.meta.env.VITE_API_URL}/api/vehicles`);
```

## 2. API 모듈 분리

컴포넌트 안에 API 호출 함수까지 같이 있으면 코드가 복잡해진다.  
`src/api/carapi.ts` 파일로 함수를 분리하고 컴포넌트에서는 import해서 호출만 한다.

```ts
// src/api/carapi.ts
import { CarResponse } from '../types'
import axios from 'axios'

export const getCars = async (): Promise<CarResponse[]> => {
  const response = await axios.get(`${import.meta.env.VITE_API_URL}/api/vehicles`);
  return response.data._embedded.cars;
}

export const deleteCar = async (url: string): Promise<void> => {
  await axios.delete(url);
}
```

## 3. React Router

### 설치
```bash
npm install react-router-dom
```

### 정의
URL을 기반으로 특정 컴포넌트를 렌더링할지 여부를 정의하는 방식.

### 주요 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| `BrowserRouter` | 웹 기반 라우터. 전체를 감싸는 최상단 컴포넌트 |
| `Routes` | Route들을 묶는 컨테이너 |
| `Route` | path와 일치하면 element로 지정한 컴포넌트를 렌더링 |
| `Link` | 페이지 이동 링크. `<a>` 태그 대신 사용 |
| `Outlet` | 중첩 라우트(Nested Routes)의 자식 컴포넌트가 렌더링될 위치 지정 |

### 기본 사용법

```tsx
// App.tsx
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'
import Home from './components/Home'
import Contact from './components/Contact'
import PageNotFound from './components/PageNotFound'

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to='/'>Home</Link>{' | '}
        <Link to='/contact'>Contact</Link>
      </nav>
      <Routes>
        <Route path='/' element={<Home />} />
        <Route path='/contact' element={<Contact />} />
        <Route path='*' element={<PageNotFound />} />
      </Routes>
    </BrowserRouter>
  )
}
```

> `path='*'` → 정의되지 않은 모든 경로를 잡아주는 404 페이지용.  
> Spring Boot SecurityConfig의 `/**` 와 동일한 개념.

---

### 중첩 라우트 (Nested Routes)

```tsx
// App.tsx — Route 중첩
<Route path='/contact' element={<Contact />}>
  <Route path='seoul' element={<ContactSeoul />} />
  <Route path='busan' element={<ContactBusan />} />
</Route>
```

```tsx
// Contact.tsx — Outlet으로 자식 컴포넌트 위치 지정
import { Link, Outlet } from 'react-router-dom'

export default function Contact() {
  return (
    <>
      <h3>Contact Component</h3>
      <nav>
        <Link to='/contact/seoul'>서울 지점</Link>{' | '}
        <Link to='/contact/busan'>부산 지점</Link>
      </nav>
      <hr />
      <Outlet />  {/* 여기에 ContactSeoul 또는 ContactBusan이 렌더링됨 */}
    </>
  )
}
```

> `Outlet` = 자식 컴포넌트가 들어올 자리표시자(Placeholder).  
> `/contact/seoul` 접근 시 Contact 컴포넌트 안의 Outlet 위치에 ContactSeoul이 렌더링된다.

---

### 와일드카드 경로

```tsx
<Route path='/contact/*' element={<Contact />} />
```
`/contact/1`, `/contact/kim` 등 하위 경로 전부를 Contact 컴포넌트로 연결할 때 사용.

## 5. DELETE 요청

### 흐름
1. `carapi.ts`에 `deleteCar()` 함수 정의
2. `useMutation`으로 DELETE 요청 처리
3. `onSuccess`에서 `invalidateQueries`로 목록 자동 갱신
4. `Snackbar`로 삭제 완료 토스트 메시지 표시

```ts
// carapi.ts
export const deleteCar = async (url: string): Promise<void> => {
  await axios.delete(url);
}
```

```tsx
// Carlist.tsx
const { mutate } = useMutation({
  mutationFn: deleteCar,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['cars'] });
    setOpen(true); // Snackbar 열기
  },
  onError: (err) => {
    console.error(err);
  },
});
```

### 삭제 후 자동 갱신 원리

```
삭제 버튼 클릭
  → mutate(url) 실행
    → deleteCar() 호출 (DELETE 요청)
      → onSuccess 실행
        → invalidateQueries(['cars']) → useQuery 자동 재실행
          → DataGrid 자동 갱신
```

> `useEffect(()=>{}, [cars])` 방식도 가능하지만,  
> React Query를 쓰고 있다면 `invalidateQueries`가 훨씬 깔끔하다.

### 삭제 버튼 renderCell

```tsx
{
  field: 'delete',
  headerName: '',
  sortable: false,
  filterable: false,
  disableColumnMenu: true,
  renderCell: (params: GridCellParams) => (
    <button
      onClick={() => {
        if (confirm(`${params.row.brand}의 ${params.row.color} ${params.row.model}을(를) 삭제하시겠습니까?`)) {
          mutate(params.row._links.self.href);
        }
      }}
    >
      삭제
    </button>
  )
}
```

### Snackbar (토스트 메시지)

```tsx
<Snackbar
  open={open}
  autoHideDuration={2000}
  onClose={() => setOpen(false)}
  message='차량 정보가 삭제 되었습니다. 🗑️'
/>
```

| prop | 설명 |
|------|------|
| `open` | 표시 여부 (boolean) |
| `autoHideDuration` | 자동으로 닫히는 시간 (밀리초) |
| `onClose` | 닫힐 때 실행할 함수 |
| `message` | 표시할 메시지 |

## 6. Carlist.tsx 최종 코드 구조 (권장 순서)

```tsx
export default function Carlist() {
  // 1. useState
  const [open, setOpen] = useState(false);

  // 2. useQueryClient
  const queryClient = useQueryClient();

  // 3. useQuery (GET)
  const { data, error, isSuccess } = useQuery({ ... });

  // 4. useMutation (DELETE)
  const { mutate } = useMutation({ ... });

  // 5. columns (mutate 참조하므로 반드시 useMutation 아래에)
  const columns: GridColDef[] = [ ... ];

  // 6. 조건부 렌더링 → return
  if (!isSuccess) {
    return <span>Loading...</span>
  } else if (error) {
    return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다.</span>
  } else {
    return (
      <>
        <DataGrid
          rows={data}
          columns={columns}
          disableRowSelectionOnClick={true}
          getRowId={row => row._links.self.href}
        />
        <Snackbar
          open={open}
          autoHideDuration={2000}
          onClose={() => setOpen(false)}
          message='차량 정보가 삭제 되었습니다. 🗑️'
        />
      </>
    )
  }
```

## 8. 자주 발생하는 오류 & 실수 모음
 
### 환경 변수 관련
 
**변수명이 `VITE_`로 시작하지 않는 경우**
```
undefined
```
- Vite는 보안상 `VITE_`로 시작하는 변수만 클라이언트에 노출한다.
- `API_URL=http://localhost:8080` 처럼 작성하면 `import.meta.env.API_URL`이 `undefined`로 나온다.
 
**`.env` 파일 위치가 잘못된 경우**
- 반드시 프로젝트 **루트 폴더**에 위치해야 한다. `src` 안에 넣으면 인식 안됨.
 
**`.env` 수정 후 반영이 안되는 경우**
- Vite 개발 서버를 **재시작**해야 환경 변수가 반영된다.
- 수정 후 `npm run dev` 다시 실행할 것.
 
---
 
### React Router 관련
 
**`BrowserRouter` 밖에서 `Link` / `useNavigate` 사용**
```
Error: useHref() may be used only in the context of a <Router> component
```
- `Link`, `Route`, `useNavigate` 등은 반드시 `BrowserRouter` 안에 있어야 한다.
- 현재 코드처럼 `BrowserRouter`와 `Routes`가 분리되어 있으면 이 오류가 발생할 수 있다.
 
```tsx
// ❌ BrowserRouter와 Routes가 분리된 경우
<>
  <BrowserRouter>
    <nav>...</nav>
  </BrowserRouter>
  <Routes>  {/* BrowserRouter 밖에 있어서 오류 */}
    <Route ... />
  </Routes>
</>
 
// ✅ Routes도 BrowserRouter 안에 있어야 함
<BrowserRouter>
  <nav>...</nav>
  <Routes>
    <Route ... />
  </Routes>
</BrowserRouter>
```
 
**`Outlet` 없이 중첩 라우트 사용**
- 부모 컴포넌트에 `<Outlet />`이 없으면 자식 라우트 컴포넌트가 아무것도 렌더링되지 않는다.
- 화면에 아무것도 안 나온다고 라우트 설정을 의심하기 전에 `Outlet` 누락 여부를 먼저 확인할 것.
 
**`path` 슬래시(`/`) 중복**
```tsx
// ❌ 중첩 라우트에서 슬래시 중복
<Route path='/contact' element={<Contact />}>
  <Route path='/seoul' element={<ContactSeoul />} />  // 앞에 / 붙이면 안됨
</Route>
 
// ✅ 자식 Route는 슬래시 없이
<Route path='/contact' element={<Contact />}>
  <Route path='seoul' element={<ContactSeoul />} />
</Route>
```

### useMutation / DELETE 관련
 
**`columns`를 `useMutation` 위에 정의**
- `mutate`가 선언되기 전에 `columns`에서 참조하는 구조가 된다.
- 동작은 되지만 안전성과 가독성을 위해 반드시 `useMutation` 아래에 정의할 것.
- **권장 순서**: `useState` → `useQueryClient` → `useQuery` → `useMutation` → `columns`
 
**`invalidateQueries` 누락**
```tsx
// ❌ invalidateQueries 없으면 삭제 후 화면이 안 바뀜
onSuccess: () => {
  setOpen(true);  // Snackbar만 뜨고 목록은 그대로
}
 
// ✅ invalidateQueries로 목록 자동 갱신
onSuccess: () => {
  queryClient.invalidateQueries({ queryKey: ['cars'] });
  setOpen(true);
}
```
 
**`queryKey`가 useQuery와 불일치**
```tsx
// useQuery의 queryKey
queryKey: ['cars']
 
// invalidateQueries의 queryKey
queryClient.invalidateQueries({ queryKey: ['car'] })  // ❌ 오타로 다른 키
```
- 키가 다르면 캐시 무효화가 안 되어 목록이 자동 갱신되지 않는다.
- `useQuery`와 `invalidateQueries`의 `queryKey`가 **정확히 일치**하는지 확인할 것.
 
---
 
### MUI DataGrid 관련
 
**`getRowId` 누락**
```
Warning: Each child in a list should have a unique "key" prop
```
- DataGrid는 기본적으로 `id` 필드를 row의 고유값으로 사용한다.
- 데이터에 `id` 필드가 없으면 `getRowId`로 고유값을 직접 지정해야 한다.
```tsx
getRowId={row => row._links.self.href}
```
 
**`console.log(data)` 미삭제**
- 개발 중 확인용으로 넣은 `console.log`는 완성 전에 반드시 제거할 것.
- 배포 환경에서 민감한 데이터가 콘솔에 노출될 수 있다.
